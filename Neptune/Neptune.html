Ubuntu/ROS/Kilted

Raw

  • Operating System & ROS2: Use Ubuntu as the base OS for your robotics development environment. Install ROS2 (Robot Operating System 2) and Kilted, depending on your project requirements.
  • Simulation Platforms: Choose a simulator that best fits your needs:
    • Nvidia Isaac Sim (preferred for advanced GPU features)
    • Webots
    • Gazebo (QT version recommended for GUI support)
  • GPU Acceleration: Ensure your system or virtual machine supports GPU acceleration. For cloud-based development, consider using an Azure VM with a minimum GPU specification. Example: Azure VM (name: class) costs approximately $500/month. Alternatively, use a local machine that meets the minimal GPU requirements for your chosen simulator.
    • Note: Applications will run without a GPU, but some features may be disabled or perform suboptimally.
    • Update your documentation with the minimum GPU requirements for each simulator.
  • Project Selection: Select an existing project that closely matches your desired end result. If no suitable project exists, create a new one tailored to your requirements.
  • Environment Import: Import relevant simulation environments or assets into your chosen simulator to accelerate development.
  • Development Tools: Install Visual Studio Code and the Quantum Development Kit (QDK) extension for enhanced coding and debugging capabilities.
  • Project Initialization: Start by initializing your project using the createproj command, a simulator-specific template, or manual setup. For full-stack development, consider using the Aspire Full Stack template with a .NET backend, or alternatives such as Node.js, Django, or Flask for web services.
    • Supported languages: C#, C++, Python, Q#, JavaScript, TypeScript, Go, Rust
    • Alternative templates: ROS2 workspace, Gazebo plugin starter, Webots Python controller, Nvidia Isaac Sim sample projects
    • Version control: Initialize a Git repository and set up remote hosting (GitHub, Azure DevOps, GitLab)
    • Configure CI/CD pipelines for automated builds and tests (GitHub Actions, Azure Pipelines, GitLab CI)
  • VS Code Extensions: In Visual Studio Code, search for and install the Quantum Development Kit (QDK) extension from Microsoft. Also consider installing:
    • ROS extension for ROS2 development
    • Python, C++, and C# language support
    • Docker and Kubernetes extensions for container orchestration
    • Remote Development extension for cloud or VM-based coding
    • GitLens for enhanced Git integration
  • Admin Portal Setup: Create an admin portal to manage users, projects, and resources. This portal can be reused for future projects.
    • NGINX Docs for web server setup
    • Kubernetes Docs for container orchestration
    • Kubernetes Dashboard for cluster management
    • Grafana for monitoring and visualization
    • Bitnami Helm Charts (Github), Bitnami Helm Charts (Enterprise), OPA (Gatekeeper Policies) for policy enforcement
    • Alternative options: Use Docker Compose for simpler setups, or managed Kubernetes services (AKS, EKS, GKE)
    • Set up authentication (OAuth, LDAP, SSO) and role-based access control
  • Reference Projects: Review and link to existing projects for inspiration or reuse.
    • Spot Project
    • Assess project objectives: Sometimes a full robot is not required; plan accordingly to optimize resources and avoid overspending.
    • Budgeting and billing: Track costs for hardware, cloud resources, and software licenses.
    • Alternative: Explore open-source robotics projects on GitHub, ROS2 demos, or Nvidia Isaac Sim samples.
  • Sensor Integration: Plan for integration of sensors such as Lidar, Sonar, Radar, and cameras.
    • Document the location and mounting of sensory units
    • Evaluate sensor compatibility with your simulator and hardware
    • Consider alternative sensors (IMU, GPS, depth cameras)
  • Hardware Placement: Design the chassis and plan hardware placement for bolt-on installation once the chassis is built.
    • Use CAD tools (Fusion 360, SolidWorks, FreeCAD) for design
    • Prototype with 3D printing or laser cutting
    • Document mounting points and cable routing
  • Optics & AI Integration: Integrate optics (cameras, depth sensors) with AI for object detection and scene understanding.
    • Use frameworks like OpenCV, TensorFlow, PyTorch, or Nvidia DeepStream
    • Test AI models in simulation before deploying to hardware
    • Alternative: Use pre-trained models or cloud-based AI services
  • Controller Connectivity: Connect and configure controllers (joysticks, gamepads, custom interfaces).
    • Test controller input in simulation
    • Configure ROS2 or simulator plugins for controller support
    • Alternative: Use web-based or mobile control interfaces
  • Testing & Validation: Develop unit, integration, and system tests for your robotics application.
    • Automate testing with CI/CD pipelines
    • Use simulation for early validation before hardware deployment
    • Document test cases and expected outcomes
  • Documentation & Collaboration: Maintain comprehensive documentation and collaborate using tools like GitHub, Azure DevOps, or Confluence.
    • Document setup instructions, architecture, and troubleshooting steps
    • Use markdown, wikis, or static site generators (MkDocs, Docusaurus)
    • Alternative: Record video walkthroughs or interactive tutorials
